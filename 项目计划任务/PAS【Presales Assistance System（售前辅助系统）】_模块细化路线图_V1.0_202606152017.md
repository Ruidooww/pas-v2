# 一、模块总览与依赖关系

## 1.1 模块分层

```
┌─────────────────────────────────────────────────────┐
│ Layer 3: 增长层（扩展业务价值）                        │
│   M8 多渠道 │ M11 经营驾驶舱 │ M12 渠道管理           │
├─────────────────────────────────────────────────────┤
│ Layer 2: 业务层（核心业务闭环）                        │
│   M3 会议场景 │ M4 合同与财务 │ M5 售后维保           │
│   M10 Agent与编排                                     │
├─────────────────────────────────────────────────────┤
│ Layer 1: 核心层（售前主链路）                          │
│   M1 售前辅助 │ M2 商机获取                            │
├─────────────────────────────────────────────────────┤
│ Layer 0: 基座层（所有模块的依赖）                      │
│   M6 AI能力 │ M7 AI知识库 │ M9 权限控制 │ M13 可扩展  │
│   M14 方案生成引擎 │ M15 客情系统(CIP)                │
└─────────────────────────────────────────────────────┘
```

## 1.2 依赖关系

| 模块 | 上游依赖 | 说明 |
|------|---------|------|
| M1 售前辅助 | M6+M7+M9+M14 | 需要AI能力+知识库+权限+方案引擎 |
| M2 商机获取 | M6+M7+M15 | 需要AI提取+知识匹配+客情数据 |
| M3 会议场景 | M6+M7+M1 | 需要AI实时+知识库+方案输出 |
| M4 合同与财务 | M6+M9 | 需要AI审核+文档权限 |
| M5 售后维保 | M6+M7 | 需要AI问答+知识库 |
| M8 多渠道 | M6+M9 | 需要AI接口+渠道权限 |
| M10 Agent与编排 | M6+M9+M13 | 需要AI+权限+可扩展框架 |
| M11 经营驾驶舱 | 所有模块 | 汇总数据，最后建设 |
| M12 渠道管理 | M9+M1 | 需要权限隔离+方案生成 |
| M14 方案生成引擎 | M6+M7 | 核心引擎，优先攻克 |
| M15 客情系统 | M6+M7 | 需要AI处理+知识匹配 |

## 1.3 细化阶段总排期

| 阶段 | 模块 | 细化周期 | 目标 |
|------|------|---------|------|
| **Phase 0** | M6+M7+M9+M13+M14 | 2-3周 | 基座模块功能规格完成 |
| **Phase 1** | M1+M2+M15 | 2-3周 | 核心业务链路功能规格完成 |
| **Phase 2** | M3+M4+M5+M10 | 2-3周 | 业务扩展功能规格完成 |
| **Phase 3** | M8+M11+M12 | 1-2周 | 增长模块功能规格完成 |

> 每个阶段内模块可并行细化，阶段间有串行依赖
